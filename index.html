My Thoughts on Progressive Enhancement

                        
Progressive enhancement focuses on the content instead of the webpage. All webpages need content and so we need to make sure that content is presented well. That means instead of testing for browsers, we test to make sure our content provides what we want to our users.

                        
We work with it in layers of different styling and code. It's important as most webpages nowadays are far more efficient and faster than older web browsers. This means that we should focus primarily on the content as no one wants content to be presented to them poorly.

                        
Codewise this means we need to work with HTML and CSS primarily while only using Javascript to improve the experience. A good experience should be provided without javascript already.

Final Project Revision and Reflection

                        
Most of my changes in the revision happened because I simply wanted to make it look better. While through this course I learned that simplicity is the best, if you oversaturate your site no one wants to view it.

                        
I also wanted to use as little javascript as possible for my work, as you should only use it when absolutely necessary to improve content the user already wants. I took the time to look at 'webkit' which helps with my extra little flair I wanted to add. Though the downside to this is that the customization doesn't work on as many browsers now.

                        
The reason I do this is because of my game design philosophy(I love game design, it's my biggest inspiration). My philosophy is that as along as your content is good, and you pay attention to the details, people will want to use what you make. That's how I'll continue to move forward with all of my programming and design.
